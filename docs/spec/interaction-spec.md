# Every Idea Counts — 交互方案细化（v2.0 两模式版）

目的：把关键交互"说死"，让实现阶段不需要再做产品判断。

适用范围：
- PWA MVP
- 单用户（仅你登录）
- 两种捕获模式：快速记录（默认）/ 深入孵化
- 想法类型手动选择：product / creative / research

> **状态**: ✅ 已实现（Phase 1-2 完成）

---

## 1) 交互基调（v2.0）

- **快**：快速记录从打开到生成笔记不超过 30 秒
- **简**：深入模式问题固定，前端静态，无 AI 调用
- **控**：用户可选择快速或深入，不被强迫
- **追**：笔记生成后可继续深入挖掘

---

## 2) 术语与 UI 文案规范

- 想法（Idea）：一条记录
- 快速记录（Quick Capture）：一键生成笔记
- 深入孵化（Deep Incubation）：3问题 + 生成笔记
- 继续深入挖掘（Continue Digging）：笔记后自由对话
- 生成笔记（Generate Note）：把输入映射为 Markdown
- 归档（Archive）：从活跃流中移出，但可搜索

按钮文案统一：
- 主动作：动词 + 结果，例如 `记录想法` / `生成笔记` / `继续深入挖掘`
- 次动作：短词，例如 `跳过` / `保存` / `归档`

## 3) Capture（/capture）交互细化

### 3.1 页面结构

```
┌─────────────────────────────────────┐
│  [快速记录] [深入孵化]               │  ← 标签切换
├─────────────────────────────────────┤
│ 想法类型: [产品] [创作] [研究]      │  ← 类型选择器
│                                     │
│ ┌─────────────────────────────────┐ │
│ │ 输入你的想法...                  │ │
│ │                                 │ │
│ └─────────────────────────────────┘ │
│                                     │
│ [辅助提示文案]                       │
│                                     │
│ [深入模式问题区 - 仅深入模式显示]    │
│                                     │
│     [记录想法 →]                     │  ← 主按钮
└─────────────────────────────────────┘
```

### 3.2 标签切换

**快速记录（默认）**：
- 显示：输入框 + 类型选择 + 主按钮
- 隐藏：深入模式问题区

**深入孵化**：
- 显示：输入框 + 类型选择 + 3个问题 + 主按钮
- 按钮文案变为：`生成笔记`

### 3.3 类型选择器

- 默认选中：`product`
- 三个选项横向排列，点击切换
- 选中状态：边框高亮

### 3.4 输入框

**placeholder（随类型切换）**：
- product：`描述你的想法...比如：一个帮助用户记录心情的App`
- creative：`描述你的创作灵感...比如：一篇关于未来城市的科幻短篇`
- research：`描述你的研究想法...比如：调查远程工作对生产力的影响`

**辅助提示（输入框下方）**：
- product：`产品想法通常涉及解决特定问题或满足用户需求。`
- creative：`创作想法可以是写作、艺术、音乐或其他形式的创意表达。`
- research：`研究想法通常涉及探索某个主题、验证假设或收集数据。`

### 3.5 深入模式问题区

**显示条件**：切换到"深入孵化"标签后显示

**问题内容（随类型切换）**：

```
    ┌─────────────────────────────────┐
    │                                 │
    └─────────────────────────────────┘

    ┌─────────────────────────────────┐
    │                                 │
    └─────────────────────────────────┘

    ┌─────────────────────────────────┐
    │                                 │
    └─────────────────────────────────┘
```

#### 创作想法

```
Q1: 主题或核心信息是什么？
    ┌─────────────────────────────────┐
    │                                 │
    └─────────────────────────────────┘

Q2: 目标受众是谁？为什么感兴趣？
    ┌─────────────────────────────────┐
    │                                 │
    └─────────────────────────────────┘

Q3: 有什么参考作品或灵感来源？
    ┌─────────────────────────────────┐
    │                                 │
    └─────────────────────────────────┘
```

#### 研究想法

```
Q1: 想探索或验证什么问题？
    ┌─────────────────────────────────┐
    │                                 │
    └─────────────────────────────────┘

Q2: 打算用什么方法研究？
    ┌─────────────────────────────────┐
    │                                 │
    └─────────────────────────────────┘

Q3: 已有相关研究有哪些？
    ┌─────────────────────────────────┐
    │                                 │
    └─────────────────────────────────┘
```

### 3.6 主按钮状态

**快速记录模式**：
- 文案：`记录想法`
- 启用条件：输入框非空
- 点击行为：创建想法 → 调用 AI → 跳转到详情页显示笔记

**深入孵化模式**：
- 文案：`生成笔记`
- 启用条件：输入框非空 + 3个问题全部填写
- 点击行为：创建想法 → 调用 AI → 跳转到详情页显示笔记

### 3.7 草稿策略

- 本地自动保存：每 1-2 秒 debounce
- 草稿内容：类型 + 输入框 + 问题答案（如有）
- 草稿恢复提示：`已恢复上次未提交内容`（Toast，3 秒）
- 提交后清空草稿

### 3.8 离线策略

- 离线时按钮仍可点击
- 点击后显示 Banner：`当前离线：想法已保存到本地，联网后将自动同步。`
- 跳转到 Library 页面（不跳转到详情页）

## 4) Idea Detail（/idea/:id）交互细化

### 4.1 页面结构

```
┌─────────────────────────────────────┐
│ Header                              │
│ ├── 标题（可编辑）                   │
│ ├── 类型 badge + 状态 badge         │
│ ├── 更新时间                        │
│ └── 状态切换按钮                     │
├─────────────────────────────────────┤
│ 笔记区                              │
│ ├── [保存] [生成笔记] [继续深入挖掘] │
│ ├── Markdown 编辑器                 │
│ └── 实时预览                        │
├─────────────────────────────────────┤
│ 导出区                              │
│ └── [导出 Markdown] [导出 HTML]     │
└─────────────────────────────────────┘
```

### 4.2 Header

**标题**：
- 默认显示：raw_input 前50字符
- 点击可编辑
- 为空时显示：`（未命名）`

**Badge**：
- 类型：`产品` / `创作` / `研究`
- 状态：`草稿` / `孵化中` / `已完成` / `已归档`
- 未同步时显示：`未同步`（橙色）

**状态切换**：
- `标记为孵化中`
- `标记为完成`
- `归档`

### 4.3 笔记区

#### 4.3.1 工具栏

**按钮**：
- `保存`：保存当前编辑内容
- `生成笔记`：调用 AI 生成笔记（追加模式）
- `继续深入挖掘`：打开对话界面

**生成笔记行为**：
- Loading：`生成中...`
- 成功：追加到笔记末尾，Toast `笔记已生成`
- 失败：Toast `生成失败，请重试`

#### 4.3.2 编辑器

- 左侧：Markdown 编辑区
- 右侧：实时预览区
- 使用 `react-markdown` + `remark-gfm` + `rehype-sanitize`

#### 4.3.3 导出

- `导出 Markdown`：下载 `.md` 文件
- `导出 HTML`：下载 `.html` 文件

### 4.4 继续深入挖掘界面

**触发**：点击"继续深入挖掘"按钮

**界面结构**：
```
┌─────────────────────────────────────┐
│ 继续深入挖掘                        │
├─────────────────────────────────────┤
│ 对话历史                            │
│ ├── AI: 你想深入了解哪方面？        │
│ ├── 用户: 我想了解竞品情况          │
│ └── AI: ...                        │
├─────────────────────────────────────┤
│ ┌─────────────────────────────────┐ │
│ │ 输入你的问题...                  │ │
│ └─────────────────────────────────┘ │
│ [发送] [更新笔记] [关闭]            │
└─────────────────────────────────────┘
```

**对话规则**：
- AI 回答最多 3 句
- AI 不反问
- AI 不改变笔记内容（除非用户点击"更新笔记"）

**更新笔记行为**：
- 将对话内容发送给 AI
- AI 生成新内容追加到笔记末尾
- Toast `笔记已更新`

## 5) Library（/library）交互细化

### 5.1 搜索与筛选

- 搜索框 placeholder：`搜索想法、笔记...`
- 默认筛选：状态 = `draft + incubating + completed`（不含 archived）
- archived 单独开关：`显示已归档`

### 5.2 卡片信息层级

- 第一行：标题 + 类型 badge + 状态 badge
- 第二行：笔记摘要（final_note 第一行或 raw_input）
- 底部：`更新于 {relative_time}`
- 右上角：未同步 badge（如有）

### 5.3 快捷动作

- 点击卡片：进入 `/idea/:id`
- 快捷按钮：`继续深入挖掘`（进入详情页并打开对话）
- 更多菜单：`归档` / `删除`

---

## 6) Weekly Review（/review）交互细化

### 6.1 默认内容

- 只展示：`draft + incubating`
- 默认排序：最久未更新（updated_at 最早优先）

### 6.2 每条 item 动作

- `继续深入挖掘`
- `生成笔记`
- `归档`

### 6.3 顶部总结

- `本周新增: {n}`
- `已完成: {n}`
- `最久未更新: {n}`

---

## 7) 错误处理

### 网络错误

- 离线提示：`当前离线，部分功能不可用`
- 重连后自动同步

### AI 生成失败

- Toast：`生成失败，请重试`
- 提供"重试"按钮

### 保存失败

- Toast：`保存失败`
- 本地缓存保留，下次重试

---

## 8) 键盘快捷键

- `Enter`：发送（在输入框中）
- `Shift+Enter`：换行
- `Esc`：关闭对话界面

---

## 9) Toast 规范

- 成功：绿色背景，显示 2 秒
- 失败：红色背景，显示 3 秒，提供操作按钮
- 提示：蓝色背景，显示 2 秒
